# Roadmap

This roadmap reflects current intentions rather than guaranteed milestones.

---

# Phase 1 — Foundation

Goal:

Establish the core architecture.

Tasks:

- [ ] bootstrap command
- [ ] manifest loading
- [ ] content loader
- [ ] progress tracking
- [ ] configuration management
- [ ] synchronization logic
- [ ] validator runner

---

# Phase 2 — Exercise Execution

Goal:

Complete the learner workflow.

Tasks:

- [ ] exercise discovery
- [ ] validation execution
- [ ] progress persistence
- [ ] status reporting
- [ ] exercise completion tracking

---

# Phase 3 — Content Authoring

Goal:

Make exercise creation easier.

Tasks:

- [ ] content templates
- [ ] exercise generator
- [ ] metadata validation
- [ ] manifest generation

---

# Phase 4 — Feedback System

Goal:

Provide more useful learning feedback.

Tasks:

- [ ] targeted hints
- [ ] error codes
- [ ] validator-specific guidance
- [ ] learning objective reporting

---

# Phase 5 — Localization

Goal:

Improve multilingual support.

Tasks:

- [ ] Spanish content
- [ ] translation tooling
- [ ] language fallback strategy

---

# Phase 6 — Advanced Validation

Goal:

Support richer exercise types.

Tasks:

- [ ] custom validators
- [ ] AST pattern matching
- [ ] style validation
- [ ] type-checking rules

---

# Future Ideas

These are exploratory and not currently planned.

- SQLite progress storage
- LLM-generated interactive hints (personalized tutoring fallback when validation fails)
- Plugin system
- Remote content distribution
- Community exercise packs
- Achievement system
- Web interface
- LSP integration
- Editor extensions


---

# Non-Goals

At least for now:

- cloud accounts
- online synchronization
- user tracking
- mandatory internet access
- complex server infrastructure

The project should remain usable entirely offline.
